System Requirements Ambiguity Analysis (SRAA)




SRAA

SRAA ย่อมาจาก System Requirements Ambiguity Analysis แปลเป็นภาษไทยว่า ระบบวิเคราะห์ความกำกวมของความต้องการระบบมีหน้าที่ตรวจจับคำกำกวมของ Requirement ที่อยู่ใน Software Requirement Specification (SRS) เวอร์ชั่นภาษาไทย โดยการใส่ข้อความที่เป็นตัวอักษรภาษาไทยในกล่องข้อความและกดปุ่ม Detected สีเขียว ระบบจะแสดงคำที่มีความกำกวมออกมาในรูปแบบสีต่างๆตามประเภทของคำกวม
